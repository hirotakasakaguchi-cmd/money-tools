# 社会保険シミュレーター現行計算ベースライン

この文書は、GitHub Issue #1のPhase 1に先立ち、現行の
`calculateSocialInsurance`が採用している入力、概算、返却値の意味を記録するものです。

記載内容とキャラクタリゼーションテストは、現在の実装を変更から守るための
**legacy behaviorのベースライン**です。制度上の正確性を証明するものではなく、
個別ケースに対する法的判断でもありません。

## 対象と今回の非対象

対象：

- 現行計算結果のGolden値による固定
- 現行計算の前提と既知の注意点の記録
- 将来の変更が現行結果へ与える差分の検出

非対象：

- 社会保険料率や税制定数の更新
- 計算式、丸め、境界処理の修正
- UI、フォーム状態、型定義の変更
- 賞与、手当、複数勤務先への対応
- 社会保険の加入可否や扶養可否の判定

## 現行入力モデル

- 年齢区分は「39歳以下」「40〜64歳」「65歳以上（参考）」の3区分
- 現在は時給、週労働時間、加入状況、配偶者手当を入力
- 変更後は週労働時間、加入予定、任意の変更後時給を入力
- 変更後時給が未指定、0、または負数の場合は現在時給を引き継ぐ
- 加入状況は`dependent`または`insured`を利用者が選択し、加入要件は判定しない

## 現行計算の前提

### 給与と月収

- 年間給与は「時給 × 週労働時間 × 52週」
- 月収は年間給与の12分の1
- 賞与と各種手当は給与に含めない

### 社会保険料

- 定数に記録された令和7年度・福岡県の概算料率を使用
- 健康保険と厚生年金は、月収から選んだ標準報酬月額に料率と12カ月を乗じる
- 雇用保険は年間給与に定率を乗じる
- `dependent`の場合、全社会保険料と標準報酬月額を0とする
- 39歳以下の介護保険料は0
- 40〜64歳と65歳以上には同じ介護保険料概算を適用する
- 65歳以上については、徴収方法が異なる旨の参考値コメントを追加する

### 税金

- 給与所得控除は現行の定数表と`rateMinus`式をそのまま使用
- 所得税は一律5%、住民税は一律10%の概算
- 基礎控除は所得税48万円、住民税43万円で固定
- 社会保険料全額を課税所得の概算から控除
- 復興特別所得税、均等割、各種所得控除、調整控除、端数処理などは扱わない

### 配偶者手当

- 現在側の「手当あり」と月額を、現在・変更後の両シナリオへ渡す
- 各シナリオが`dependent`の場合だけ、月額の12カ月分を手取りへ加える
- 変更後が`insured`の場合は手当が0になるため、条件に応じて消滅コメントを出す
- 会社ごとの支給条件は判定しない

### 厚生年金メリット

- `insured`の場合、月収に報酬比例部分の概算率と12カ月を乗じる
- 現在と変更後の年間増加見込みの差を算出する
- 「90歳までの総リターン」は年間差額の25年分で固定する

### 労働時間とコメント

- 年間増加時間は「max(0, 変更後週時間 − 現在週時間) × 52週」
- 年間増加時間が0の場合、1時間あたり現金リターンは`null`
- 1時間あたり現金リターンのコメント区分は100円未満、500円未満、それ以上
- コメントは加入状況、配偶者手当、手取り差、年金差、年齢区分から決定する

## Golden値として固定する既知の注意点

以下は制度上の正しさとは別に、現行挙動として意図的にテストへ記録しています。

- 浮動小数点の未丸め値を返す
- 65歳以上にも40〜64歳と同じ介護保険料概算を適用する
- 給与所得控除の`rateMinus`境界直後で控除額が65万円の下限へ戻る場合がある
- 変更後の配偶者手当を独立入力できず、現在側の設定を再利用する
- 明示した変更後時給0円も「未指定」と同じ扱いになる
- 標準報酬月額は実際の算定基礎ではなく、年間給与の12分の1から選ぶ
- 税額は単一税率の簡易概算で、税制上の正確な税額ではない
- 加入状況は入力値を前提とし、週労働時間などから加入可否を判定しない

これらを修正する場合は、制度・仕様変更用の別テストを追加し、Golden値との差分を
レビューしたうえで更新します。キャラクタリゼーションテストを、実装に合わせて
理由なく書き換えてはいけません。

## テスト実行

```bash
pnpm test
pnpm test:coverage
pnpm lint
pnpm build
```

`pnpm test`と`pnpm test:coverage`は非対話モードで終了するため、CIからそのまま実行できます。
Vitest設定ではTypeScriptと`@/`パスエイリアスを利用できます。

現時点ではリポジトリ内にCI設定とNodeバージョン固定ファイルはありません。
ローカルおよびCIでは、Next.js、Vitest、ViteがサポートするNode.jsバージョンを
使用する必要があります。
